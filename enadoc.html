Case Study

Redesigning the Upload and Capture workflow

An improved way of adding documents to Enadoc for faster performance and better experience.

Background

Enadoc, a browser-based document imaging and management solution, has recently added two new features: My Workspace and Smart Tags. Both presenting a couple of challenges, mainly:

A gap in the user's workflow

The user now has to go to a different page to add a document to My Workspace, and another one to upload it to Enadoc.

Poor performance

Consequently, the software has to load and reload the document preview a total of three times.

The Challenge

To create an optimized and intuitive workflow that lets the user add documents to My Workspace, tag them, and upload them to Enadoc.

A Problem and Three Prototypes

This problem unwittingly came up in the middle of a 3-day internal meeting overseas — perfect timing, but a slightly limited duration for exploration (considering we had other things in the agenda).

A suggestion that was agreed upon is inspired by Microsoft Azure’s horizontal scrolling pattern:





However, the current workflow is not linear. After capturing a document, a user has to go back to My Workspace in order to tag it. We found a way to solve this problem later on, but at this point we stuck with the Azure-style navigation.

By editing existing screens and creating a quick animation on Principle, we managed to define the step-by-step flow and explore possible transitions.

As I began redesigning the screens, it became clear that the tagging layout had to be changed as well in order to adapt to the new navigational flow. Some of the changes are:

TABS INSTEAD OF STEP-BY-STEP TASKS

Since not every step is required, it will be easier for the user to navigate with tabs instead of clicking skip/next for every action. The user can easily go back and forth without much delay. Additionally, the program will not have to reload and preview the document for every step.

A MODAL FOR CHOOSING A TAG PROFILE

- Size constraints (tag cloud + properties + default values)
- Better conceptual model for the user because even if it is in tag cloud form, nakahiwalay siya sa tag cloud ng tagging
- ngayon nasa breadcrumbs sya para malaman na under tag profile ung tags na nakikita nya ngayon
