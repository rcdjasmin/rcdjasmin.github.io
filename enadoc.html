Case Study

Redesigning the Upload/Capture Workflow

An improved way of adding documents to Enadoc for faster performance and better experience.

Background

Enadoc, a browser-based document imaging and management solution, has recently added two new features: My Workspace and Smart Tags. Both present a couple of UX challenges, mainly:

A gap in the user's workflow

The user has to go to a different page to add a document to My Workspace, and another to upload it to Enadoc.

Poor performance

As a consequence, the software has to load and reload the document preview a total of three times.

The Challenge

To create an optimized and intuitive workflow that lets the user add documents to My Workspace, tag them, and upload them to Enadoc.

Problems & Prototypes

After brainstorming, we all agreed on a solution inspired by Microsoft Azure’s horizontal scrolling pattern. Here, each step is revealed in a panel that animates sideways. There will be four panels: My Workspace, Document Preview, Capture, and the simplified Tag/Redact.

Now the user can go through the entire process without ever having to leave the page.

But there is one setback: the current workflow is not linear. After capturing a document, the user has to go back to My Workspace and wait for the Smart Tags (generated via optical character recognition) before proceeding to the next step.

As I began redesigning the screens, it became clear that the tagging's navigational layout had to be restructured as well. Some of the changes are:

TABS INSTEAD OF STEP-BY-STEP NAVIGATION

Since not every step is required, we switched to tabs instead of clicking skip or next for every action. Now the user can easily go back and forth without much delay. Additionally, the document preview won't have to reload for every step.

A MODAL FOR CHOOSING A TAG PROFILE

The step of choosing a tag profile cannot fit into one panel, thus, a resizable modal is introduced. This also eliminates the problem of confusing this step with the tagging step (both of which have almost identical tag clouds).

It seemed we were ready for development. However, after conducting a quick usability test with our colleagues, we had to make a couple more changes:

Adding 'Edit mode' in Capture Settings

The buttons to save the capture settings has been replaced by an 'Edit' link because the buttons distracted the users from their main goal.

Giving the user the freedom to choose where to go next

To accommodate different behaviors, a prompt asking the user how to proceed is added.
